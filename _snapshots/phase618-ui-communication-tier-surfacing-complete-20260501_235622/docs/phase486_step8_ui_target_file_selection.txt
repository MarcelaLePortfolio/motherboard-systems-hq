PHASE 486 — STEP 8
UI TARGET FILE SELECTION

────────────────────────────────

OBJECTIVE

Select the exact UI render host for initial governance trace panel implementation
using the refined shortlist from Step 7.

This is a selection step only.
No UI mutation occurs in this step.

────────────────────────────────

INPUT BASIS

Source:
docs/phase486_step7_refined_ui_surface_shortlist.txt

Selection criteria:

• Existing operator-facing render surface
• Lowest coupling risk
• Closest to live demo visibility path
• No backend mutation required
• Suitable host for passive governance trace rendering

────────────────────────────────

SELECTED TARGET FILE

app/demo-runtime/page.tsx

────────────────────────────────

SELECTION REASON

app/demo-runtime/page.tsx is the correct first render host because:

• It is an existing UI page surface
• It is already aligned to demo/runtime visibility
• It is operator-facing
• It avoids backend mutation
• It can consume existing contracts passively
• It is safer than choosing transport, contract, reducer, or builder files

────────────────────────────────

EXPLICIT NON-SELECTIONS

The following classes are NOT valid initial render hosts:

• src/cognition/transport/*
• src/contracts/*
• src/governance_operator_*/*
• src/governance_investigation/*
• app/api/*
• reducers
• builders
• adapters
• runtime pipeline files

Reason:

These are not passive UI render hosts.
Choosing them would increase coupling risk or violate UI-only scope.

────────────────────────────────

IMPLEMENTATION BOUNDARY

Allowed next step:

• Minimal passive render addition inside app/demo-runtime/page.tsx

Still prohibited:

• Backend mutation
• Contract mutation
• Governance logic movement
• Approval logic movement
• Execution logic movement
• Async introduction
• Multi-request expansion

────────────────────────────────

COMPLETION CONDITION

Target file selection is complete when:

• Exact UI host is named
• Non-UI files are excluded
• Next step can proceed with single-boundary mutation

────────────────────────────────

CLASSIFICATION

UI HOST SELECTION COMPLETE
