FAILURE CLASSIFICATION MODEL

Category 1 — Human Interpretation Failures

Description:
Operator misunderstands governance signals, approvals, or system posture.

Examples:

Misreading approval state
Confusing readiness with authorization
Misinterpreting telemetry severity
Assuming visibility equals permission

Risk:

Premature execution pressure
Unsafe assumptions
Operator confusion

Detection Signals:

Conflicting operator commands
Repeated clarification requests
Governance approval bypass attempts
Policy interpretation conflicts

Mitigation Expectations:

Clear governance state indicators
Explicit approval states
Human-readable governance summaries
Operator confirmation checkpoints

────────────────────────────────

Category 2 — Policy Ambiguity Failures

Description:

Governance rules unclear or open to interpretation.

Examples:

Undefined approval boundaries
Incomplete validation definitions
Conflicting governance documents

Risk:

Policy drift
Inconsistent decisions
Approval disagreement

Detection Signals:

Multiple valid interpretations
Inconsistent approval outcomes
Governance clarification loops

Mitigation Expectations:

Single-source governance doctrine
Policy versioning
Conflict detection markers
Doctrine clarity reviews

Category 3 — Operator Load Failures
