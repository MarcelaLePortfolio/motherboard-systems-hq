REQUEST PANEL PASSIVE CONSUMPTION SPEC

OBJECTIVE

Define a safe, UI-only refinement of the Request panel
using existing report fields without introducing transformation.

────────────────────────────────

TARGET PANEL

Request

Current fields:

• report.requestId
• report.requestSummary

────────────────────────────────

REFINEMENT GOAL

Improve structural clarity of the Request panel by exposing
the request summary with stronger separation between identifier
and prompt content while preserving exact backend values.

This is visibility refinement only.

────────────────────────────────

FIELD USAGE (EXACT)

  report.requestId

• Natural-language prompt:
  report.requestSummary

────────────────────────────────

PLACEMENT

Inside existing Request panel block:

Request
• Request ID: existing direct render
• Natural-language prompt: existing direct render

Optional passive refinement allowed:

• Stronger section labeling
• Readability-only spacing adjustments
• Multi-line prompt presentation if needed

────────────────────────────────

NO-TRANSFORMATION RULE

UI must NOT:

• summarize requestSummary
• shorten requestSummary
• derive request categories
• infer request intent labels
• add computed request metadata

UI must only:

• render report.requestId directly
• render report.requestSummary directly

────────────────────────────────

NO-DATA RULE

No additional NO DATA handling required.

Fields are always present in report contract.

────────────────────────────────

COMPLETION CONDITION

Spec is complete when:

• exact fields are named
• exact placement is defined
• no-transformation rule is defined
• implementation can proceed as one UI-only mutation
