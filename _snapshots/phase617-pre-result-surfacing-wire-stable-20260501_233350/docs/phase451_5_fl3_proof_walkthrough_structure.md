Objectives
Tasks

Constraints

STAGE 3 — GOVERNANCE EVALUATION EXPOSURE

Safety considerations
Execution risks
Authority boundaries
Governance reasoning

Invariant:

Governance reasoning must be visible.

STAGE 4 — APPROVAL REQUIREMENT EXPOSURE

Execution cannot proceed automatically.

Approval boundary

Invariant:

STAGE 5 — EXECUTION STRUCTURE EXPOSURE

System demonstrates:

Execution sequence
Task boundaries

Failure paths

Invariant:

Execution must remain bounded.

STAGE 6 — REPORTING STRUCTURE EXPOSURE

System demonstrates:

Outcome reporting structure
Trace continuity
Operator visibility

Invariant:

Operator must remain informed.

STAGE 7 — TRACE CONTINUITY EXPOSURE

System demonstrates:

Full reasoning continuity from request
through structured preparation.

Invariant:

No reasoning gaps.
