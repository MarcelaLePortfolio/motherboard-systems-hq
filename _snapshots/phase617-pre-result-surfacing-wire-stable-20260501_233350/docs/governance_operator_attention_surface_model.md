# Phase 402.2 — Governance Operator Attention Surface Model

## Purpose

Define how governance cognition highlights what deserves operator attention without creating priority queues, execution influence, or routing behavior.

This defines attention signaling capability only.

Not execution influence.

---

## Capability Objective

The operator must be able to quickly see:

What changed
What is stable
What deserves awareness
What can be safely ignored

This reduces operator scanning effort.

---

## Attention Surface Definition

Attention surfaces must provide:

Change indicators
Stability indicators
Investigation suggestions
Signal grouping
Noise suppression

This is cognition ergonomics.

Not task prioritization.

---

## Change Indicators

Must identify:

New cognition results
Trust state changes
Interpretation changes
New investigation context

Change indicators must be:

Deterministic
Evidence-based
Non-alarmist

No urgency semantics allowed.

---

## Stability Indicators

Must identify:

No change detected
Trust stable
Interpretation unchanged
Signals consistent

Operators must be able to trust stability signals.

Silence must mean stability.

---

## Investigation Suggestions

May indicate:

Evidence worth reviewing
Trust shifts
Historical divergence
Explanation updates

Suggestions must never imply required action.

Only optional inspection.

---

## Signal Grouping

Attention signals may be grouped by:

Trust category
Governance interpretation class
Investigation context
Signal origin

Grouping improves cognition scanning.

No execution meaning allowed.

---

## Noise Suppression Principle

Attention surfaces must avoid:

Signal spam
Repeated unchanged signals
Low-value noise

Cognition clarity is the objective.

---

## Explicit Prohibitions

Attention surfaces must never:

Create priority queues
Imply urgency
Trigger execution
Reorder tasks
Influence routing

Attention is visibility only.

---

## Safety Guarantees

This phase introduces:

No runtime behavior
No execution coupling
No routing logic
No reducer wiring
No automation interaction

Capability definition only.
