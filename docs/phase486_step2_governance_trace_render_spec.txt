PHASE 486 — STEP 2
GOVERNANCE TRACE RENDER SPEC (CONSUMPTION-ONLY)

────────────────────────────────

OBJECTIVE

Define exact UI render contract for governance trace panel
without introducing logic, transformation, or coupling.

This is a STRUCTURAL RENDER SPEC — not implementation.

────────────────────────────────

INPUT CONTRACT (READ-ONLY)

UI must consume existing governance trace contract as-is.

Assumed shape (do not modify source):

• rules[] (ordered)
  - ruleId
  - evaluationResult (MATCH / NO_MATCH)
  - reason (string)
  - priority (implicit via order)

UI must NOT:
• infer missing fields
• compute values
• reorder rules

────────────────────────────────

RENDER STRUCTURE

UI must render:

GOVERNANCE TRACE PANEL

For each rule (in order):

[INDEX] Rule: {ruleId}
Status: {evaluationResult}
Reason: {reason}

Separator between rules

────────────────────────────────

ORDERING REQUIREMENT

• Must strictly follow backend order
• No sorting
• No grouping
• No filtering

Order = truth

────────────────────────────────

DISPLAY RULES

evaluationResult must be displayed exactly:

• MATCH
• NO_MATCH

No substitution, no styling reinterpretation

────────────────────────────────

REASON VISIBILITY

• Must display full reason string
• No truncation
• No summarization
• No transformation

────────────────────────────────

PROHIBITED BEHAVIOR

UI must NOT:

• Collapse rules
• Hide non-matching rules
• Highlight based on custom logic
• Derive conclusions
• Summarize results
• Introduce computed fields

────────────────────────────────

ALLOWED BEHAVIOR

UI MAY:

• Add static labels (e.g., "Governance Trace")
• Add visual separators
• Use basic formatting for readability

ONLY if:
• It does not alter meaning
• It does not introduce logic

────────────────────────────────

VALIDATION CHECKS

Render is VALID if:

• Rule order matches backend exactly
• All rules displayed
• All reasons displayed in full
• No computed values exist
• No backend mutation required

────────────────────────────────

FAILURE CONDITIONS

Render is INVALID if:

• Any rule missing
• Any rule reordered
• Any reason truncated
• Any logic introduced
• Any data transformed

────────────────────────────────

CLASSIFICATION

This is:

READ-ONLY CONTRACT RENDER SPEC

NOT:

• UI enhancement
• UX optimization
• Data processing layer
