Assumption
Memory

Explicit
Current

Time-bound

Authority confirmation requirements:

1 — EXPLICIT HUMAN INITIATION

Every request must originate from a human operator.

Invalid origins:

System generated
Agent generated
Worker generated
Derived request chains

Failure condition:

Non-human origin → INVALID REQUEST

2 — FRESH AUTHORITY REQUIREMENT

Authority must be current.

Concept examples:

Authority timestamp
Session confirmation
Operator reconfirmation

Failure condition:

Stale authority → INVALID REQUEST

3 — SCOPE AUTHORITY MATCH

Operator authority must match request scope.

Concept examples:

Operator domain
Service ownership
Agent responsibility

Failure condition:

Authority mismatch → INVALID REQUEST

4 — NON-TRANSFERABLE AUTHORITY

Authority cannot be inherited automatically.

Explicit prohibition:

Agent delegation of authority.

Failure condition:

Derived authority → INVALID REQUEST

5 — AUTHORITY REVOCATION POSSIBILITY

Authority must always be revocable.

Concept examples:

Operator cancel ability
Governance override
Authority withdrawal

Failure condition:

Irrevocable authority → NOT SAFE
